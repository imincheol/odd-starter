# 📝 Report: 설치 스크립트 UX 개선 (초기화 vs 업데이트 분기 안내)

## 1. Summary of Work (작업 요약)
- **환경 감지 로직 추가**: `install.sh` 실행 시 로컬의 `ATLAS.md` 존재 여부를 확인하여 신규 설치(New)와 시스템 업데이트(Update)를 자동으로 판별하는 로직을 추가했습니다.
- **상황별 동선 안내**: 설치 완료 메시지를 분기 처리하여, 신규 사용자에게는 `ODD_INIT.md`를, 기존 사용자에게는 `ODD_UPDATE.md`를 AI에게 전달하도록 명확한 행동 지침을 제공합니다.
- **템플릿 동기화**: 수정된 `install.sh` 로직을 `odd-template`에도 동일하게 적용하여 배포 무결성을 유지했습니다.

## 2. Key Changes (주요 변경 사항)

### 📂 install.sh (v0.9.3)
- `IS_UPDATE` 변수를 도입하여 프로젝트 환경을 사전에 스캔.
- 완료 메시지에 시각적 구분(🆕 vs 🔄) 및 맞춤형 가이드 파일 경로 명시.

### ✅ 검증 테스트 결과
- **업데이트 시나리오**: 기존 프로젝트에서 실행 시 "[업데이트 완료]" 메시지와 함께 `ODD_UPDATE.md`가 안내되는 것을 확인했습니다.
- **안내 문구**: 단순히 "규칙을 확인하라"는 메시지에서 "AI에게 가이드를 전달하라"는 액션 중심의 메시지로 개선되었습니다.

## 3. Review & Conclusion (검토 및 결론)
- 이번 개선을 통해 사용자는 ODD 시스템 설치 후 "그래서 이제 뭘 해야 하지?"라는 물음에 대한 즉각적인 답을 얻을 수 있게 되었습니다. 
- AI와의 협업 시작점이 명확해짐에 따라 도구 사용의 편의성이 크게 향상될 것으로 기대됩니다.
